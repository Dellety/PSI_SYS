# 备件订货交付跟踪系统 - 开发任务书

> 

---

## 一、项目概述

### 1.1 项目背景

我司作为代理商，承接工厂客户的备件采购与交付业务。当前业务流程涉及多个角色、多个环节的协作，缺乏统一的信息化系统进行全链路跟踪。为提升工作效率、降低信息传递风险、实现全流程可追溯，需建设一套**备件订货交付跟踪系统**。

### 1.2 项目目标

1. **全流程可视化**：从工厂提出备件需求到客户签收归档，所有环节在系统中有完整记录
2. **角色协同**：销售、项目负责人、采购员三方在系统中高效协作，信息实时同步
3. **数据沉淀**：供应商信息、客户信息、物料库、采购记录、合同信息统一管理
4. **预警提醒**：货期超期、加急订单等关键节点自动预警通知
5. **决策支撑**：通过报表看板为管理层提供数据支撑

### 1.3 系统定位

- **系统类型**：Web 应用系统（浏览器访问）
- **用户规模**：内部使用，预计 10-50 人
- **部署方式**：私有化部署（预留 OA 对接能力）

- **采购员**：面向供应商的采购接口人，可查看采购价但不可见客户报价
- **系统管理员**：仅负责系统维护，不参与业务流程操作

## 三、业务流程设计

### 3.1 正向流程（11 个环节）

```
┌─────────────────────────────────────────────────────────────────┐
│                        正向业务流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ① 工厂提出备件需求                                              │
│         │                                                       │
│         ▼                                                       │
│  ② 客户向我方销售提出备件询价                                     │
│         │                                                       │
│         ▼                                                       │
│  ③ 项目负责人与工厂客户及我方客户确认采购内容                        │
│         │                                                       │
│         ▼                                                       │
│  ④ 客户向我方销售提出正式采购询价                                   │
│     → 销售报价                                                   │
│     → 签订合同（线下签订，系统录入）                                │
│         │                                                       │
│         ▼                                                       │
│  ⑤ 销售邮件通知项目负责人采购下料（邮件草稿→人工确认发送）           │
│         │                                                       │
│         ▼                                                       │
│  ⑥ 项目负责人向采购员提交采购订单（采购员自行拆单）                  │
│         │                                                       │
│         ▼                                                       │
│  ⑦ 采购员向供应商采购                                             │
│         │                                                       │
│         ▼                                                       │
│  ⑧ 采购到货 → 通知项目负责人验收                                   │
│         │                                                       │
│         ▼                                                       │
│  ⑨ 验收通过 → 项目询问销售发货地址 → 销售确认地址                  │
│     → 项目发货 → 将发货单/快递单号发送给销售                       │
│     → 销售将快递单号同步给客户                                     │
│         │                                                       │
│         ▼                                                       │
│  ⑩ 客户收货 → 反馈签字版收货单                                    │
│     → 项目负责人归档收货单 → 同步给销售                             │
│         │                                                       │
│         ▼                                                       │
│  ⑪ 销售关闭该项目                                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 逆向流程（退换货）

```
销售发起退换货申请
        │
        ▼
项目负责人确认退换货
        │
        ▼
通知采购员换货/退货
        │
        ▼
采购员向供应商执行换货/退货
        │
        ▼
换货到货后走正常到货验收→发货流程
```

### 3.3 订单状态流转

```
待确认内容 → 待报价 → 待签合同 → 待下料
→ 待采购 → 采购中 → 待验收 → 验收中
→ 待发货 → 已发货 → 待签收 → 已签收 → 已关闭

异常分支：
任意环节 → 退换货中 → 退换货完成 → 回到原流程或关闭
任意环节 → 已取消
```

### 3.4 发货地址规则

- 客户档案中维护多个收货地址（工厂地址、客户自身地址等）
- 每次发货前，项目负责人向销售确认发货地址
- 销售从客户已有地址中选择或输入临时地址
- 地址确认后锁定，作为该订单的发货凭证

#### 4.7.3 订单状态机定义

```python
ORDER_STATUS = {
    # 正向流程
    "pending_confirm":    "待确认内容",      # 项目负责人确认采购内容
    "pending_quote":     "待报价",           # 等待销售报价
    "pending_contract":  "待签合同",         # 等待合同签订
    "pending_dispatch":   "待下料",           # 合同签订后待通知下料
    "pending_purchase":  "待采购",           # 待采购员采购
    "purchasing":        "采购中",           # 采购员已下单
    "pending_inspect":   "待验收",           # 采购到货待验收
    "inspecting":        "验收中",           # 验收进行中
    "pending_ship":      "待发货",           # 验收通过待发货
    "shipped":           "已发货",           # 已发货
    "pending_receipt":    "待签收",           # 待客户签收
    "received":          "已签收",           # 客户已签收
    "closed":            "已关闭",           # 项目关闭

    # 异常流程
    "return_exchange":   "退换货中",         # 退换货流程
    "cancelled":         "已取消",           # 订单取消
}

# 状态流转规则
STATUS_TRANSITIONS = {
    "pending_confirm":    ["pending_quote", "cancelled"],
    "pending_quote":     ["pending_contract", "cancelled"],
    "pending_contract":  ["pending_dispatch", "cancelled"],
    "pending_dispatch":   ["pending_purchase"],
    "pending_purchase":  ["purchasing", "cancelled"],
    "purchasing":        ["pending_inspect", "return_exchange"],
    "pending_inspect":   ["inspecting"],
    "inspecting":        ["pending_ship", "return_exchange"],
    "pending_ship":      ["shipped"],
    "shipped":           ["pending_receipt", "return_exchange"],
    "pending_receipt":    ["received"],
    "received":          ["closed"],
    "return_exchange":   ["pending_inspect", "cancelled"],  # 换货完成后回到验收
}
```

#### 4.8.2 通知流程

```
业务事件触发
    │
    ▼
生成邮件草稿（自动填充收件人、主题、正文）
    │
    ▼
邮件草稿列表展示给对应操作人
    │
    ▼
操作人审核草稿内容，确认发送
    │
    ▼
系统发送邮件，记录发送日志
```

#### 4.8.3 邮件模板字段

每封邮件需包含：
- 订单编号
- 客户名称
- 备件明细摘要
- 当前状态
- 加急标记（如有）
- 相关链接（跳转到订单详情）
